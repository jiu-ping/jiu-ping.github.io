background

  • background-color：设置元素的背景颜色；

  • background-image： url(‘URL’)：设置元素的背景图像；

  • background-repeat：控制背景图像是否重复；

    repeat 默认值，背景图像将在垂直方向和水平方向上重复
    repeat-x 背景图像仅在水平方向上重复
    repeat-y 背景图像仅在垂直方向上重复
    no-repeat 背景图像仅显示一次，不在任何方向上重复

  • background-position :默认相对于左上角

  • ackground-origin 是 CSS3 中新增的属性。
    在使用 background-position 属性来设置背景图像的位置时，默认是以元素左上角的位置来计算的。
    您还可以使用 background-origin 属性来设置 background-position 属性相对哪个位置来定位背景图像

    值 描述
    padding-box 相对于元素的内边距区域来定位背景图像
    border-box 相对于元素的边框区域来定位背景图像
    content-box 相对于元素的内容区域来定位背景图像

  • background-attachment 属性用来设置背景图像是固定在某个位置还是跟随页面一起滚动

    值 描述
    scroll 默认值，背景图像随着页面元素的滚动而移动
    fixed 当页面的其余部分滚动时，背景图像固定不动

  • background-size： x% y%
    使用百分比表示背景图像相对于所在元素宽度与高度的百分比，x% 表示宽度，y% 表示高度，如果只设置第一个值，那么第二个值将被设置为默认值 auto（自动）

  • background-clip ： 是 CSS3 中新增的属性，通过它可以设置背景图像的显示区域。

    值 描述
    border-box 默认值，在元素边框及以内的区域显示背景图像
    padding-box 在元素内边距及以内的区域显示背景图像
    content-box 在元素内容区域显示背景图像

  • *简写**：background 是背景属性的简写形式
    color-image-position-size-repeat-attachment-origin-clip

    css表格样式

    表格样式属性：table-layout border-collapse border-spacing caption-side empty-cells

    • table-layout 定义表格样式
    格式 描述
    automatic 默认值，自定表格布局，表示表格中每列的宽度视单元格中的内容而定
    fixed 固定表格布局，表示表格的宽度由列宽度、单元格边框、单元格之间的间距等因素而定
  1. 固定表格布局
    固定表格布局允许浏览器更快地对表格进行布局。在固定表格布局中，表格的水平宽度仅取决于列宽度、表格边框宽度、单元格间距等因素，与单元格中的内容无关。也就是说，表格中超出表格宽度的内容可能会被忽略。
  2. 自动表格布局
    在自动表格布局中，列的宽度视单元格中的内容（没有换行的最宽内容）而定，也就是说如果某个单元格的宽度为 100px，但单元格中内容所占据的宽度要大于 100px，这就会导致单元格中的内容将单元格撑大。正是因为这一特点，此种算法可能会比较慢。

  • border-collapse 用来设置是否合并表格中相邻的边框

    格式 描述
    separate 默认值，相邻的两个边框是分开的，使用它不会忽略 border-spacing 和 empty-cells 属性
    collapse 相邻的两个边框会合并为一个单一的边框，使用它会忽略 border-spacing 和 empty-cells 属性

  • border-spacing 可以设置相邻单元格边框之间的距离（border-collapse：separate 时才有效）

  • caption-side 可以设置表格标题的位置

    格式 描述
    top 默认值，将表格标题定位在表格正上方
    bottom 将表格标题定位在表格正下方

  • empty-cells 设置当某个单元格中没有内容时，是否显示这个空单元格 （border-collapse：separate 时才有效）

    格式 描述
    hide 隐藏空单元格周围的边框
    show 默认值，显示空单元格周围的边框

margin

为元素设置外边距

外边距折叠

外边距折叠指的是相邻的两个或多个外边距会在垂直方向上发生合并，合并为一个外边距。关于外边距折叠有以下几点需要注意：
margin 折叠只发生在块级元素上；
1.浮动元素的外边距不会与任何外边距发生折叠；
2.设置了 overflow 属性且值不为 visible 的块级元素，将不会与它的子元素发生外边距折叠；
3.绝对定位元素的外边距不与任何外边距发生折叠；
4.根元素（例如）的外边距不与其它任何外边距发生折叠。

在相邻的两个兄弟元素，父元素与子元素之间：
如果相邻两个元素外边距的值都为正数，那么两个元素的实际间距为两个外边距中较大的那个；
如果相邻两个元素外边距的值都为负数，那么两个元素的实际间距为两个外边距中较小的那个；
如果相邻两个元素外边距的值一个为正数、一个为负数，那么两个元素之间的实际间距为两个外边距相加的和。

css计数器

使用计数器三步走：
1.counter-reset：none | [ ]
阻止复位 计数器名字 计数器起始值
2.counter-increment:格式同上
3.counter(name) 

    body   {  
            counter-reset: chapter;   
            }  
    h5, h6 {  
              margin: 5px 0 5px;  
            }  
    h5     {  
             counter-reset: section;   //创建在该标签之后调用的计数器
             counter-increment: chapter;  //为本标签中调用计数器设置增加属性
            }  
    h6     {  
             counter-increment: section;  
            }  
    h5:before {  
           content: "Chapter " counter(chapter) ". ";  
            }  
    h6:before {  
           content: counter(chapter) "." counter(section) " ";  
            }

counters(name, string, list-style-type)
计数器名称 连接嵌套与被嵌套计数器的字符 计数器风格 

     ol {
        /* 为每个ol元素中li创建新的计数器实例 */
        counter-reset: ol-list;
        list-style-type: none;
    }
    li:before {
        /* 只增加计数器的当前实例 */
        counter-increment: ol-list;
        /* 为所有计数器实例增加以“.”分隔的值 */
        content: counters(ol-list, ".") "、";
    }

position

css中定位有五种方式：

  • static 静态定位 元素显示在正常位置
  • relative 相对定位 相对于正常位置

  • absolute 绝对定位 相对于第一个非静态父元素

  • fixed 相对于浏览器
  • sticky 粘性定位，它是 relative 和 fixed 的结合体，能够实线类似吸附的效果，当滚动页面时它的效果与 relative 相同，当要滚动到屏幕之外时则会自动变成 fixed 的效果。

css float

浮动可以使一个元素脱离自己原本的位置，并在父元素的内容区中向左或向右移动，直到碰到父元素内容区的边界或者其它浮动元素为止。
另外，在浮动元素之后定义的文本或者行内元素都将环绕在浮动元素的一侧

  • 添加浮动：left right none(默认值)
  • 清除浮动：left right both none(默认值)
